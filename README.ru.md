# pos-register

[🇬🇧 English](README.md) · **🇷🇺 Русский** · [🇰🇿 Қазақша](README.kk.md)

---

Касса с открытым исходным кодом для розничной торговли в Казахстане.
Работает на Windows, Linux, macOS, Android и iOS — офлайн по умолчанию,
синхронизируется с облаком, когда есть сеть.

Написана на Flutter. Бесплатно использовать, свободно изучать, свободно
форкать. Лицензия Apache 2.0.

---

## Что это такое?

`pos-register` — это приложение кассы, которое работает на терминале.
Поддерживает сканирование, взвешивание, скидки, разделение оплаты,
продажу в долг, фискальные чеки и ежедневный цикл открытия / закрытия
смены с X- и Z-отчётами.

Это часть небольшой семьи открытых POS-приложений для магазинов в
Казахстане (и везде, где подходят те же шаблоны — НДС 12 % или 0 %,
тенге, каталог НТИН / ХТИН):

| Репозиторий | Что делает |
|---|---|
| **pos-register** _(этот репозиторий)_ | приложение кассира на терминале |
| [pos-server](https://github.com/icybeard/pos-server) | облачная часть — .NET 10 + PostgreSQL 16, мультитенантность с row-level security |
| [pos-admin](https://github.com/icybeard/pos-admin) | веб-админка (товары, склад, платёжный календарь, отчёты) |
| [pos-shared](https://github.com/icybeard/pos-shared) | контракты обмена — protobuf-схемы, общие для всех трёх |

Каждый репозиторий — настоящее работающее приложение, которое можно
склонировать прямо сейчас. Касса полностью запускается сама с локальной
SQLite-базой, если хочется попробовать до того, как поднимать облако.

## Для владельцев магазинов и бухгалтеров

Не обязательно быть разработчиком, чтобы пользоваться этим. Первый релиз
выходит с небольшой, но достаточной админкой — чтобы вести магазин
одного формата от начала до конца: товары, цены, кассиры, смены, чеки,
базовый список остатков и платёжный календарь. Более крупные возможности
(перемещения между магазинами, учёт партий и сроков годности, рецепты
для кофеен, прайс-листы поставщиков и т. д.) уже есть в коде и выходят
в ближайших релизах — карта развития в каталоге
[`specs/`](https://github.com/icybeard/pos-docs/tree/main/specs)
репозитория [pos-docs](https://github.com/icybeard/pos-docs).

Если хочется попробовать на реальном железе, самый короткий путь такой:

1. Поставить Flutter на ноутбук. ([Как установить Flutter →](https://docs.flutter.dev/get-started/install))
2. Выполнить три команды из раздела **Для разработчиков** ниже.
3. Касса откроется на экране с пустой корзиной и без товаров.
4. Чтобы появились товары, цены и кассиры, дополнительно запустите
   pos-server + pos-admin. README в [pos-server](https://github.com/icybeard/pos-server)
   проведёт по облачной части.

Если хочется, чтобы кто-то настроил всё за вас, — проект молодой, кнопки
«купить хостинг» пока нет, она в планах. А пока код — ваш: любой
разработчик может развернуть весь стек из этого репозитория и трёх
соседних.

## Для разработчиков

```bash
git clone --recurse-submodules https://github.com/icybeard/pos-register.git
cd pos-register
flutter pub get
flutter run -d macos    # или: linux / windows / chrome / <устройство>
```

Это весь быстрый старт — без сервера, без Postgres, без Docker. Касса
работает поверх локальной SQLite-базы (через
[drift](https://drift.simonbinder.eu/)) и полноценно используется
офлайн. Локальная база — источник истины на устройстве; `sync_outbox`
постепенно сливается на центральный сервер, когда тот доступен.

> **Предупреждение про первую сборку.** Первый `flutter pub get` (и
> первая сборка на каждой платформе) компилирует ~13 МБ исходников
> SQLite3 Multiple Ciphers из
> [`third_party/sqlite3mc/`](third_party/sqlite3mc/). Это занимает
> ~60–90 с на каждую целевую тройку, дальше всё кешируется в
> `.dart_tool/hooks_runner/`. Мы вендорим исходники, чтобы сборки были
> самодостаточными — без зависимости от того, доступен ли CDN
> GitHub-релизов. Подробности — в
> [`third_party/sqlite3mc/README.md`](third_party/sqlite3mc/README.md).

### Запуск против локального сервера

1. Запустите [pos-server](https://github.com/icybeard/pos-server) (его
   README описывает Postgres на `:5433` и API на `:5000`).
2. Запустите `pos-register`.
3. В [pos-admin](https://github.com/icybeard/pos-admin) откройте
   **Касса → Активировать** и привяжите устройство к арендатору +
   workstation id. Касса начнёт подтягивать товары, кассиров и смены
   через sync.

### Перегенерация proto-стабов

```bash
dart pub global activate protoc_plugin   # один раз
make proto-gen
```

Proto-схемы лежат в [pos-shared](https://github.com/icybeard/pos-shared)
как git-сабмодуль по пути `proto/`. Меняем там, переносим указатель
сабмодуля сюда, перегенерируем.

### Тесты

```bash
make test       # unit + widget + BLoC, golden-PDF чеки
make analyze    # flutter analyze (должно быть чисто до PR)
```

### Из чего состоит

| Стек | |
|---|---|
| Flutter 3.x + Dart 3 | UI + состояние (BLoC) |
| drift (SQLite + WAL) | локальный источник истины |
| gRPC Subscribe stream | реальные пуши остатков с сервера |
| REST sync push / pull | массовая синхронизация каталога и чеков |
| Apache 2.0 | всё целиком, без условий |

Несколько фиксированных правил, которые стоит знать перед тем, как
делать PR:

- **Деньги — целочисленные тиыны** (1 ₸ = 100 тиын). Никаких float
  нигде — ни на проводе, ни в расчётах.
- **Весовой товар**: `итого = (вес_г / 1000) × цена_за_кг_тиын`.
- **Ставки НДС**: 12 % или 0 % (никогда 10 / 20). Специфика Казахстана.
- **НТИН / ХТИН**: у каждого товара должен быть код Национального
  каталога товаров; временный ХТИН действует 30 дней.
- **Offline-first**: каждая запись сначала идёт в drift. Sync сливает
  потом. Если найдёте место, где данные уходят на сервер до drift, —
  это баг.
- **Нет «главной» кассы.** Каждый терминал — тонкий клиент к облаку,
  никакая касса не зависит от другой кассы.

Полный архитектурный обзор — в [`CLAUDE.md`](CLAUDE.md) (изначально
писался как бриф для ИИ-агента, но это самый компактный обзор проекта,
который у нас есть).

## Статус проекта

Проект молодой. Что-то будет ломаться, дизайн будет меняться, первые
релизы будут опираться на технических пользователей, чтобы закрывать
бреши в админке. Мы намеренно не обещаем больше, чем готов код.

- ✅ Кассовый поток: сканирование / взвешивание / скидки / смешанная
  оплата / продажа в долг.
- ✅ Работа офлайн и синхронизация.
- ✅ Двуязычный интерфейс (русский, қазақша).
- 🟡 Админка (товары, склад, платёжный календарь, отчёты) —
  минимальная, но рабочая в v1; расширяется с каждым релизом.
- 🟡 Интеграция с НКТ (nct.kz) — работает для типовых случаев,
  шероховатости остаются.
- ⏳ Фискализация Webkassa (этап P8) — спроектирована, ещё не выпущена.
- ⏳ Сверка с банком, Telegram-уведомления, мультивалютность, выгрузка
  в 1С, ЭСФ — спроектированы, ещё не выпущены.

Дорожная карта по фичам — в [pos-docs](https://github.com/icybeard/pos-docs),
там же лог реализации.

## Как контрибьютить

PR-ы приветствуются. Самый короткий путь:

1. Откройте issue или прокомментируйте существующий, чтобы не делать
   одну и ту же работу дважды.
2. Ветка от `main`, PR фокусированный (одна фича или один фикс на PR).
3. `make analyze && make test` должно проходить чисто.
4. Любая новая строка интерфейса идёт через ARB-файлы
   (`lib/core/l10n/app_ru.arb` + `app_kk.arb`) — никакого хардкода в
   виджетах.
5. Деньги — всегда целочисленные тиыны. Всегда.

CLA нет. Вклад лицензируется под Apache 2.0, как и весь проект.
