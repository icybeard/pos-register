# pos-register

[🇬🇧 English](README.md) · [🇷🇺 Русский](README.ru.md) · **🇰🇿 Қазақша**

---

Қазақстан бөлшек саудасына арналған ашық бастапқы кодты касса. Windows,
Linux, macOS, Android және iOS жүйелерінде жұмыс істейді — әдепкі бойынша
офлайн режимде, желі қосылғанда бұлтпен синхрондалады.

Flutter-де жасалған. Тегін пайдалану, еркін қарау, еркін форк жасау
рұқсат етілген. Apache 2.0 лицензиясы.

---

## Бұл не?

`pos-register` — терминалда жұмыс істейтін касса қосымшасы. Сканерлеу,
өлшеу, жеңілдіктер, төлемді бөлу, борышқа сату, фискалдық чектер және
ауысымды ашу / жабудың күнделікті циклін X- және Z-есептермен қоса
қолдайды.

Бұл — Қазақстандағы (және ҚҚС 12 % немесе 0 %, теңге, ҰТТН / ХТТН
тауар каталогы сияқты бірдей үлгілер сәйкес келетін кез келген жерде)
дүкендерге арналған шағын ашық кодты POS-қосымшалар отбасының бір бөлігі:

| Репозиторий | Не істейді |
|---|---|
| **pos-register** _(осы репозиторий)_ | терминалдағы кассир қосымшасы |
| [pos-server](https://github.com/icybeard/pos-server) | бұлт жағы — .NET 10 + PostgreSQL 16, row-level security бар көп-тенантты |
| [pos-admin](https://github.com/icybeard/pos-admin) | веб-әкімші панелі (тауарлар, қойма, төлем күнтізбесі, есептер) |
| [pos-shared](https://github.com/icybeard/pos-shared) | алмасу келісімшарттары — үшеуіне ортақ protobuf-сұлбалары |

Әр репозиторий — қазір клондап алуға болатын нақты жұмыс істейтін
қосымша. Бұлтты көтермей тұрып, кассаны өзіңіздің жергілікті SQLite
базасымен бөлек сынап көруге болады.

## Дүкен иелері мен бухгалтерлер үшін

Бұны пайдалану үшін әзірлеуші болудың қажеті жоқ. Бірінші релиз шағын,
бірақ жеткілікті әкімші панелімен шығады — бір форматтағы дүкенді басынан
аяғына дейін жүргізу үшін: тауарлар, бағалар, кассирлер, ауысымдар,
чектер, негізгі қалдық тізімі және төлем күнтізбесі. Үлкенірек
мүмкіндіктер (дүкендер арасындағы жылжытулар, партиялар мен жарамдылық
мерзімдерін есепке алу, кофеханаларға арналған рецепттер, жеткізушілер
прайс-парақтары, т. б.) кодта бар және келесі релиздерде шығып жатыр —
жол картасы [pos-docs](https://github.com/icybeard/pos-docs)
репозиторийінің [`specs/`](https://github.com/icybeard/pos-docs/tree/main/specs)
каталогында.

Нақты жабдықта сынап көргіңіз келсе, ең қысқа жол мынадай:

1. Ноутбукке Flutter орнатыңыз. ([Flutter-ді қалай орнату керек →](https://docs.flutter.dev/get-started/install))
2. Төмендегі **Әзірлеушілер үшін** бөліміндегі үш команданы орындаңыз.
3. Касса экраныңызда бос себетпен және тауарлар жоқ күйде ашылады.
4. Тауарлар, бағалар мен кассирлер пайда болуы үшін қосымша
   pos-server + pos-admin іске қосу қажет. [pos-server](https://github.com/icybeard/pos-server)
   README-сі бұлт жағы бойынша жетекшілік етеді.

Біреудің бәрін сізге орнатып беруін қалайтын болсаңыз — жоба әлі жас,
«хостинг сатып алу» түймесі әлі жоқ, ол жоспарда. Әзірге код — сіздікі:
кез келген әзірлеуші осы репозиторий мен қалған үшеуінен бүкіл стекті
көтере алады.

## Әзірлеушілер үшін

```bash
git clone --recurse-submodules https://github.com/icybeard/pos-register.git
cd pos-register
flutter pub get
flutter run -d macos    # немесе: linux / windows / chrome / <құрылғы>
```

Бұл — толық жылдам бастау — серверсіз, Postgres-сіз, Docker-сіз. Касса
жергілікті SQLite базасының үстінде ([drift](https://drift.simonbinder.eu/)
арқылы) жұмыс істейді және офлайн режимде толық пайдаланылады. Жергілікті
дерекқор — құрылғыдағы шындықтың дереккөзі; `sync_outbox` орталық сервер
қол жетімді болған кезде біртіндеп ағады.

> **Бірінші құрастыру туралы ескерту.** Бірінші `flutter pub get` (және
> әр платформадағы бірінші құрастыру)
> [`third_party/sqlite3mc/`](third_party/sqlite3mc/) каталогынан SQLite3
> Multiple Ciphers-тің ~13 МБ бастапқы кодын құрастырады. Бұл әр мақсатты
> үштікке ~60–90 с алады, одан кейін бәрі `.dart_tool/hooks_runner/`-да
> кэштеледі. Біз бастапқы кодты вендорлаймыз, осылайша құрастырулар
> өзін-өзі қамтамасыз етеді — GitHub-релиздерінің CDN-іне тәуелсіз.
> Толығырақ —
> [`third_party/sqlite3mc/README.md`](third_party/sqlite3mc/README.md).

### Жергілікті серверге қарсы іске қосу

1. [pos-server](https://github.com/icybeard/pos-server) іске қосыңыз
   (README-сі `:5433`-тегі Postgres пен `:5000`-дағы API-ді сипаттайды).
2. `pos-register` іске қосыңыз.
3. [pos-admin](https://github.com/icybeard/pos-admin)-да **Касса →
   Белсендіру** менюін ашып, құрылғыны тенант пен workstation id-ге
   байланыстырыңыз. Касса тауарлар, кассирлер мен ауысымдарды sync
   арқылы тарта бастайды.

### Proto-стабтарды қайта генерациялау

```bash
dart pub global activate protoc_plugin   # бір рет
make proto-gen
```

Proto-сұлбалары [pos-shared](https://github.com/icybeard/pos-shared)-та
`proto/` жолы бойынша git-сабмодул ретінде жатыр. Сонда өзгертіңіз, осы
жердегі сабмодул сілтемесін жаңартыңыз, қайта генерациялаңыз.

### Тесттер

```bash
make test       # unit + widget + BLoC, golden-PDF чектер
make analyze    # flutter analyze (PR-ға дейін таза болуы керек)
```

### Не қолданылады

| Стек | |
|---|---|
| Flutter 3.x + Dart 3 | UI + күй (BLoC) |
| drift (SQLite + WAL) | жергілікті шындық дереккөзі |
| gRPC Subscribe stream | сервер жағынан нақты уақыттағы қалдық пушетері |
| REST sync push / pull | каталог пен чектердің жаппай синхрондалуы |
| Apache 2.0 | бәрі толығымен, шартсыз |

PR жасамас бұрын білуге тұрарлық бірнеше тіркелген ережелер:

- **Ақша — бүтін сандағы тиындар** (1 ₸ = 100 тиын). Ешқандай float
  ешжерде — желіде де, есептеулерде де емес.
- **Өлшенетін тауар**: `барлығы = (салмақ_г / 1000) × бір_кг_бағасы_тиын`.
- **ҚҚС мөлшерлемелері**: 12 % немесе 0 % (ешқашан 10 / 20).
  Қазақстанға тән.
- **ҰТТН / ХТТН**: әр тауардың Ұлттық тауар каталогының коды болуы
  керек; уақытша ХТТН 30 күн жарамды.
- **Offline-first**: әр жазу алдымен drift-ке түседі. Sync кейіннен
  ағады. Дерек серверге drift-тен бұрын кететін жерді тапсаңыз — бұл
  қате.
- **«Басты» касса жоқ.** Әр терминал — бұлтқа бағытталған жұқа клиент,
  ешбір касса басқа кассаға тәуелді емес.

Толық сәулеттік шолу — [`CLAUDE.md`](CLAUDE.md)-да (бастапқыда ИИ-агент
үшін бриф ретінде жазылған, бірақ бізде бар жобаның ең ықшам шолуы).

## Жобаның күйі

Жоба жас. Бірдеңелер сынады, дизайн өзгереді, алғашқы релиздер әкімші
панеліндегі олқылықтарды жабу үшін техникалық пайдаланушыларға арқа
сүйейді. Біз код әзір болмай тұрып артық уәде беруден саналы түрде
бас тартамыз.

- ✅ Кассалық ағын: сканерлеу / өлшеу / жеңілдік / аралас төлем /
  борышқа сату.
- ✅ Офлайн жұмыс және синхрондау.
- ✅ Екі тілді интерфейс (русский, қазақша).
- 🟡 Әкімші панелі (тауарлар, қойма, төлем күнтізбесі, есептер) —
  v1-де минимал, бірақ жұмыс істейді; әр релизде кеңейеді.
- 🟡 НКТ интеграциясы (nct.kz) — типтік жағдайларда жұмыс істейді,
  кедір-бұдыр жерлер қалды.
- ⏳ Webkassa фискализациясы (P8 кезеңі) — жобаланған, әлі шықпаған.
- ⏳ Банкпен салыстыру, Telegram хабарламалары, көп валюта, 1С-ке
  экспорт, ЭСФ — жобаланған, әлі шықпаған.

Жол картасын [pos-docs](https://github.com/icybeard/pos-docs)-та табуға
болады, сол жерде іске асыру журналы да бар.

## Қалай үлес қосуға болады

PR-лер құпталады. Ең қысқа жол:

1. Қос жұмыс істемеу үшін алдымен issue ашыңыз немесе бар issue-ге
   пікір қалдырыңыз.
2. `main`-нен бұтақ жасаңыз, PR-ды фокусты ұстаңыз (бір фича немесе
   бір түзету — бір PR-ға).
3. `make analyze && make test` таза өтуі керек.
4. Кез келген жаңа интерфейс жолы ARB-файлдар арқылы
   (`lib/core/l10n/app_ru.arb` + `app_kk.arb`) өтеді — виджеттерде
   хардкод болмайды.
5. Ақша — әрқашан бүтін сандағы тиындар. Әрқашан.

CLA жоқ. Үлес Apache 2.0 лицензиясымен, бүкіл жоба сияқты, лицензияланады.
